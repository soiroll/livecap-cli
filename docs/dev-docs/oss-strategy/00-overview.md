## 背景と目的

### なぜOSS化するのか

LiveCapは現在Steam上で商用販売中ですが、リリース後の開発が停滞しています。OSS化により以下を実現します：

#### 主要目標

1. **コミュニティの力を借りる**
   - 開発者コミュニティからの貢献
   - バグ報告と改善提案の増加
   - 多様なユースケースへの対応

2. **技術的信頼性の向上**
   - ソースコードの透明性
   - セキュリティ監査の容易化
   - エンタープライズ採用への道

3. **ただ乗り防止**
   - 企業の無断商用利用を防ぐ
   - 適切なライセンス管理
   - 収益は主目的ではない

### OSS化しない場合のリスク

- 開発停滞の継続
- 既存ユーザーの離脱
- 技術的負債の増加
- コミュニティ形成の機会損失

## 現状分析

### プロジェクト状況

| 項目 | 現状 | OSS化後の期待 |
|------|------|--------------|
| **開発頻度** | 低（副業、週数時間） | コミュニティ貢献で増加 |
| **ユーザー数** | Steam 350購入者 | 大幅増加見込み |
| **コミュニティ** | Discord 82人 | GitHub経由で拡大 |
| **収益** | Steam販売のみ | Steam + 商用ライセンス |
| **サポート負荷** | 管理可能 | 初期は増加、後にコミュニティ化 |

### 開発者の制約

- **時間**: 副業として週数時間程度
- **予算**: 初期投資ゼロを希望
- **対応可能範囲**: 英語メール対応OK
- **収益重視度**: 低（コミュニティ優先）

### 技術的強み

✅ すべての著作権をPine Labが保持
✅ 依存ライブラリがGPL互換（後述）
✅ Steam配信の実績あり
✅ TEN Award受賞による信頼性
✅ 先進的なAI技術（NVIDIA Parakeet, Riva等）

## 戦略の選択

### 採用する戦略：シンプルデュアルライセンス

LiveCapの状況に最適化した**軽量なデュアルライセンスモデル**を採用します：

```
LiveCap
├─ GPL v3版（無料・オープンソース）
│  ├─ GitHub完全公開
│  ├─ 個人・教育・OSS利用は完全無料
│  ├─ ソースからビルド
│  └─ コミュニティサポート
│
└─ 商用ライセンス（有料・個別対応）
   ├─ 企業の商用利用向け
   ├─ メールベースの個別対応
   ├─ 料金: $100〜$500（規模に応じて）
   └─ 自動販売システムなし（手動処理）
```

### なぜこの戦略か

#### ✅ メリット

1. **初期投資ゼロ**
   - 自動販売システム不要
   - 弁護士レビュー不要（シンプル文書）
   - プラットフォーム費用なし

2. **管理負荷が最小**
   - 商用ライセンス問い合わせは月1-2件程度と予想
   - ケースバイケースで柔軟対応
   - 自動化の複雑さを回避

3. **コミュニティ優先**
   - GPL版で最大限の自由
   - 収益化は二の次
   - ただ乗りのみ防止

#### ⚠️ 注意点

- 大規模な商用利用には不向き（数十社以上）
- 問い合わせ対応の手間（許容範囲内）
- Steam規約との整合性確認が必要

### 他の選択肢を採用しない理由

#### ❌ フル機能デュアルライセンス（Qt方式）
- 自動販売システムが必要 → 初期投資大
- サポート義務が重い → 時間制約
- 収益目的でないため過剰

#### ❌ MIT + 有料サポート
- ただ乗り防止が不十分
- 企業が無料で商用利用可能
- 目的（ただ乗り防止）を達成できない

#### ❌ 完全GPL（商用ライセンスなし）
- 企業の商用利用を制限できない
- フォーク版の商用化を防げない
- ビジネス的リスクあり
